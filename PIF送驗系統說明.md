# 依洛嘉 PIF 送驗系統說明文件

> 版本：v8.2-sync　最後種子版：2026-06-17-pif7-fixed

---

## 一、系統概覽

「依洛嘉送驗系統」是一套給**依洛嘉國際實業有限公司**（以下稱「依洛嘉」）與其**客戶**（化妝品代工委託商）共同使用的化妝品 PIF（Product Information File）送驗管理平台，以單一 HTML 檔案（index.html）形式交付，搭配 Firebase Firestore 進行雲端同步。

系統同時服務三種使用者端：

| 端口 | 說明 |
|------|------|
| **客戶端** | 客戶登入後送驗申請、查帳、追蹤進度 |
| **依洛嘉後臺** | 內部員工管理收件、開請款單、送驗資料匯出、PIF 處理 |
| **公共展示牆** | 免登入瀏覽已完成 PIF 的產品 |

---

## 二、登入身分與權限

### 客戶
- Google 帳號登入（需由依洛嘉事先開通）
- 可自行申請帳號，待依洛嘉審核後啟用
- 登入後需填寫廠商資料（中英文公司名、地址、電話）

### 依洛嘉後臺
兩種角色，功能略有差異：

| 角色 | 可用功能 |
|------|---------|
| **主要管理者** | 全部功能，含帳號管理 |
| **管理** | 全部功能，無帳號管理入口 |

後臺功能頁籤：總覽、客戶、樣品庫、收件、申請、送驗資料、帳款明細、PIF 處理狀態、新品介紹、收款匯總

### 公共展示牆
- 免登入，任何人皆可瀏覽已公開的 PIF 完成品項

## 三、送驗流程（產品狀態機）

每筆產品依序流轉以下狀態：

```
applied → approved → pending → shipped → received
→ invoiced → reported → paid → testing → reporting → completed
```

| 狀態 | 顯示文字 | 說明 |
|------|---------|------|
| `applied` | 等待依洛嘉確認 | 客戶送出申請，等待依洛嘉允許寄送 |
| `approved` | 可寄送 | 依洛嘉確認，客戶可備樣寄出 |
| `pending` | 待寄送 | 客戶已選定，準備安排寄送 |
| `shipped` | 已寄出樣品 | 客戶已寄出，依洛嘉收件中 |
| `received` | 已收樣 | 依洛嘉已收到樣品，準備開請款單 |
| `invoiced` | 待匯款 | 已開收款單，等待客戶匯款 |
| `reported` | 待查帳 | 客戶已回報匯款，依洛嘉查帳中 |
| `paid` | 已收款待送驗 | 依洛嘉確認收款，安排送驗 |
| `testing` | 送驗中 | 樣品已送往檢測中心 |
| `reporting` | PIF 製作中 | 依洛嘉製作 PIF 報告 |
| `completed` | 已完成 PIF | 可到雲端資料夾下載 PIF |

## 四、各功能頁說明

### 4.1 客戶端

#### 送驗（custSubmit）
- 查看並編輯廠商資料
- 開啟依洛嘉 PIF 共用清單（可申請共用已有 PIF 的品項）
- 開啟我的 PIF 雲端資料夾
- 新增送驗產品（手動填寫或批次匯入）
- 勾選「可寄送」品項後，一鍵安排寄送
- 已申請但未確認的品項可自行撤銷

#### 帳務（custBilling）
- 顯示合約制客戶預付餘額及扣款紀錄
- 待匯款批次：每批次顯示品項清單及合計金額
- 客戶填入「匯款末五碼或匯款人姓名」後送出匯款回報
- 可對金額有誤的帳單提出申訴

#### 進度看板（custBoard）
- 全部品項統計（進行中 / 已完成）
- 按流程階段分區顯示每筆產品當前狀態
- 快速連結我的 PIF 雲端資料夾
- 顯示依洛嘉可共用的 PIF 產品清單

#### 新品介紹（custShowcase）
- 瀏覽依洛嘉的新品介紹卡片
- 可搜尋、分頁瀏覽

---

### 4.2 依洛嘉後臺

#### 總覽（elgHome）
- 快速看板：各狀態品項數量
- 點擊可快速跳轉對應功能頁

#### 客戶（elgCustomers）
- 客戶清單：姓名、Email、登入方式、狀態
- 新增客戶（手動）、審核待審核申請
- 設定客戶 Gmail、產生驗證連結
- 查看客戶所有品項，並可清除客戶產品資料
- 合約制客戶：設定預付餘額、建立合約帳本

#### 樣品庫（elgLibrary）
- 全部品項的主資料庫
- 可依「廠商 / 狀態」篩選，支援搜尋
- 批量操作：刪除、轉移廠商
- 匯入：貼上文字（自動解析）、圖片 OCR（Tesseract.js）、上傳 XLSX
- 草稿區：辨識後先暫存、核對再確認匯入

#### 收件（elgReceive）
- 列出所有「已寄出（shipped）」的品項
- 逐筆或批量標記為「已收樣（received）」
- 可附帶備注、設定收件日期

#### 申請（elgApply）
- 列出客戶新送出的申請（applied）
- 逐筆允許（→ approved）或拒絕並刪除
- 可批量操作

#### 送驗資料（elgData）
- 選定產品，查看、編輯詳細送驗資訊：
  - 送驗廠商、地址、工廠（固定為依洛嘉）
  - 產品中英文名、別名、型號規格、數量、類型、樣品量
- 檢驗項目（依產品類型自動帶入，可調整）：
  - 防腐效能測驗（+$350）
  - 常規9項+念珠菌（+$320）
  - 穩定性測試（+$350）
- 產品類型預設邏輯：
  - 油劑 → 穩定性測試
  - 面膜 → 常規9項+念珠菌
  - 液劑霜劑 → 防腐+常規
- 一鍵生成：合作協議（Word 格式）＋ 樣品清單 Excel（ExcelJS）
  - 合作協議：依洛嘉 × 广东微化检验科技有限公司
  - 同時將品項轉為「送驗中（testing）」狀態

#### 帳款明細（elgInvoice）

**開請款單（recv）**
- 依客戶分組，勾選收樣品項
- 選 PIF 費用方案後開立請款單：

| 方案 | 金額 | 說明 |
|------|------|------|
| VIP 客戶價 | $10,000 | |
| 換名檢驗費 | $5,000 | 只換名、不重新送驗 |
| 代工客戶公定檢測費 | $10,000 | 單筆訂單10萬以上 |
| PIF 處理費（含檢測） | $20,000 | 含全套檢測 |

- 合約制客戶：餘額充足自動扣款，不足差額另開收款單
- 依洛嘉自有品項：可批量免收款

**已開單（issued）**
- 列出已向客戶開立、尚未收款的品項

**查帳（check）**
- 客戶已回報匯款，逐筆確認並標記已收款（→ paid）
- 可新增付款記錄（金額、付款方式、備注）

**補收 PIF 費（retro）**
- 針對已送驗但尚無 PIF 費收款記錄的品項補開請款單
- 支援分期：每次開立獨立收款單，不覆蓋先前紀錄
- 可手動追加品項（從樣品庫挑選或手寫新增）

**請款單查詢（query）**
- 查詢所有已建立的獨立收款單（invoiceRecs）

#### PIF 處理狀態（elgPif）
- 列出已收款待送驗以後的品項
- 可設定 PIF 方案標籤、上傳 PIF 雲端資料夾連結
- 標記 PIF 費是否已收、是否已完成

#### 新品介紹（elgShowcase）
- 新增新品卡片：名稱（中 / 英）、簡介、封面圖、介紹文件（HTML）、外部連結
- 可批量上架至公共展示牆（publicShowcase）
- 刪除、搜尋、分頁

#### 收款匯總（elgSummary）
- 全客戶應收 / 已收 / 差額統計
- 按付款方式分計
- 點擊客戶可快速跳轉到「補收 PIF 費」頁

## 五、AI 功能

### 英文產品名稱翻譯
- 呼叫 Anthropic Claude API（`claude-sonnet-4-5`）
- 根據中文品名 + 產品類型，自動生成專業英文名
- 需在 localStorage 儲存 API Key（`elg_anthropic_key`）

### OCR 圖片辨識
- 使用 Tesseract.js（繁體中文）
- 從圖片中辨識品名，批量匯入到草稿區

## 六、資料結構

### 品項（product）
```json
{
  "id": "imp1718000000001",
  "name": "玫瑰精華液",
  "alias": "內部別名",
  "en": "Rose Essence Serum",
  "spec": "50ml/瓶",
  "qty": "4瓶",
  "type": "液劑霜劑",
  "t": { "fang": true, "chang": true, "wen": false },
  "cust": "晶寶企業行",
  "mine": true,
  "status": "applied",
  "batch": "",
  "charge": 10000,
  "payRecords": [],
  "invoices": [],
  "customerEmail": "customer@gmail.com"
}
```

### 客戶（customer）
```json
{
  "id": "firestore-doc-id",
  "name": "晶寶企業行",
  "email": "customer@gmail.com",
  "login": "Google",
  "status": "已啟用",
  "drive": "https://drive.google.com/...",
  "driveAuth": true,
  "contract": false,
  "balance": 0
}
```

### 收款單（invoiceRec）
```json
{
  "id": "INV-20260625-001",
  "cust": "晶寶企業行",
  "date": "2026-06-25",
  "method": "匯款",
  "note": "訂金",
  "total": 10000,
  "status": "待收款",
  "items": [
    { "id": "...", "name": "玫瑰精華液", "plan": "VIP客戶價", "amt": 10000 }
  ]
}
```

### 帳本桶設計（雙桶分離，永不遷移）
- **檢驗費桶**：`p.charge`（應收）、`p.payRecords`（已收）— 舊資料相容
- **額外請款單桶**：`p.invoices[]`（每張 PIF 費 / 分期款獨立一張）
- 合計 = 兩桶相加，互不影響

## 七、雲端同步（Firebase）

- Firestore 集合：`products`、`customers`、`staff`、`sharedPIF`、`publicShowcase`、`showcase`
- 前臺登入：Firebase Auth（Google OAuth）
- 後臺員工：需先在 Firestore `staff` 集合建立記錄，登入後自動識別角色
- 客戶登入後讀取 `customers` 集合比對 Email，完成身分確認
- 離線模式：localStorage 持久化，可在無網路環境操作

## 十、已知設計決策

1. **單一 HTML 檔**：整個系統為一個 index.html，方便部署與更新
2. **不刪不蓋**：帳本採雙桶模型，收款記錄永不覆蓋，分期款獨立一張
3. **版本種子**：`SEED_VER` 字串控制 localStorage 資料相容性，版本不符自動清空
4. **狀態機嚴格單向**：產品狀態只往後流轉，部分狀態下客戶可申請撤銷（後臺確認）
5. **檢驗費 vs PIF 費分離**：`charge/payRecords` 為送驗成本，`invoices` 為對客戶的 PIF 服務費，兩者完全獨立計算
